## 二、接真实数据前必须补（触发条件：任何真实群记录要进系统/出现在演示中）

按优先级排序，每项标注"为什么"与"大概工作量"：

1. **数据处理告知与同意基础**（法务，非代码）
   - JING 私域：需在群规/服务协议中告知"聊天数据用于运营分析"，或对既有数据做匿名化处理（匿名化后不再属个人信息）。
   - 外部品牌 pilot：pilot 协议必须含数据处理条款（处理目的/范围/期限/删除承诺）。
   - 工作量：模板一次性起草（可让 Cindy/法务协助），每客户复用。

2. **原始导出文件不落盘策略**
   - 现状：解析器输出已脱敏，但上传的原始 txt 的存储策略未定。
   - 要求：内存中解析，解析完成即丢弃原文件；若必须暂存，加密 + 24h 自动删除。
   - 工作量：上传接口实现时半天内置。

3. **静态加密与访问控制**
   - 数据库磁盘加密（云盘加密选项）+ 应用层最小权限账号 + 管理接口鉴权（当前 demo 无鉴权）。
   - 工作量：1-2 天（含 auth 模块 MVP）。

4. **删除机制（被遗忘权）**
   - 按群/按成员代号删除全部关联数据的管理操作 + 删除日志。
   - 工作量：1 天。

5. **备份纪律**（架构文档 §5 既有要求）
   - pg_dump 定时备份 + 异地拷贝 + 恢复演练。demo 阶段可缓，真实数据后必须。
   - 工作量：半天脚本 + 例行执行。

6. **审计日志**
   - 谁在何时导入/查看/导出了哪个群的数据。投资人尽调与 PIPL 合规都会看。
   - 工作量：1 天（事件表 + 中间件）。
